## Track Format

When creating a new track:

1. **Spec**: `conductor/tracks/<track-id>/spec.md` - What and why
2. **Plan**: `conductor/tracks/<track-id>/plan.md` - How (implementation steps)
3. **Status**: Updated in this file

### Track States
- `draft` - Being defined
- `ready` - Spec approved, plan ready
- `in-progress` - Implementation started
- `review` - Implementation complete, under review
- `done` - Merged and verified
